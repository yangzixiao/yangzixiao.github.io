开发与探索笔记

发表于 2018-05-24

第一章：Activity的生命周期和启动模式

1. 生命周期

1.1 正常情况下
onCreate() -> onStart() -> onResume() -> onPause() -> onStop() -> onDestroy()

onRestart()的调用时机：
1. 开启新Activity又回到旧Activity,旧Activity会调用
2. 屏幕息屏亮屏切换亮屏时

打开新Activity时，旧Activity的onPause()和新Activity的onResume()谁先调用
旧Activity的onPause()先调用
1.2 异常情况下
1. 横竖屏切换
    1. 未配置屏幕横竖屏切换会重新创建
    onPause() -> onSaveInstanceState() -> onStop() -> onDestroy() -> onCreate() -> onStart() -> onRestoreInstanceState() ->  onResume()

    onSaveInstanceState()和onRestInstanceState()是用来保存和恢复数据
    onSaveInstanceState()和onRestInstanceState()调用时机
    onSaveInstanceState()在onStop()之前调用
    onRestInstanceState()在onStart()之后调用

    2. 配置屏幕横竖屏切换不会重新创建
    android:configChanges="orientation|keyboardHidden|screenSize"
    不会重建Activity，切换时会调用OnConfigurationChanged()方法

    注意：这个属性在低版本上是可以起作用。在android 4.0 以上不起作用，必须要加上screenSize，即android:configChanges="keyboardHidden|orientation|screenSize"

2. 内存不足时

    如果内存不足，会调用onSaveInstanceState()和onRestInstanceState()是保存和恢复数据

2. 启动模式

standard
标准模式：默认启动模式，开启一个Activity创建一个实例
singleTop
栈顶复用模式：当栈顶的Activity与要启动的Activity一样时，复用该Activity并调用onNewIntent方法
singleTask（默认具有ClearTop效果）
单任务栈模式：就是任务栈内只存在一个Activity
singleInstance
单实例模式：就是一个Activity实例单独占用一个任务栈

View的事件体系

1. 基础知识
1.1 View的定义
1.2 View的位置参数
View的位置由其四个顶点决定：top,left,right,bottom
注意：这四个顶点是相对于父布局的
3.0开始，增加了x,y,translationX,translationY几个参数
其中x,y指View的左上角坐标
translationX,translationY指View相对于父布局的偏移量

几者的换算关系是：
x = left + translationX
y = top + translationY

注意：在平移过程中，top,left,right,bottom是View的原始位置，值是不会改变的，x,y则是随着translationX,translationY的改变而改变的
1.3 MotionEvent
手势事件包含：
ACTION_DOWN,ACTION_MOVE,ACTION_UP等
获取当前时间的坐标方式：
getX()/getRawX(),getX()/getRawX()
getX(),getY()获取的是相对于该View的坐标
getRawX(),getRawX()获取的是相对于屏幕的坐标
1.4 TouchSlop
系统所能识别的最小滑动距离
获取方式：ViewConfiguration.get(context).getScaledTouchSlop();

3. View事件分发机制

3.1 主要方法

dispatchTouchEvent(MotionEvent ev)
分发-->处理点击时间的分发

onInterceptTouchEvent(MotionEvent ev)
拦截-->确定当前View是否拦截事件

onTouchEvent(MotionEvent ev)
点击-->处理事件

3.2 三者关系

dispatchTouchEvent(MotionEvent ev){
  boolean consume = false;
  if(onInterceptTouchEvent(MotionEvent ev)){
    consume = onTouchEvent(MotionEvent ev)
  }else{
    consume = child.dispatchTouchEvent(MotionEvent ev);
  }
  return consume;
}

当前View调用onDispatchTouchEvent(MotionEvent ev)分发事件，然后调用onInterceptTouchEvent(MotionEvent ev),如果返回true，说明当前View要消费事件，则会调用当前View的OnTouchEvent(MotionEvent ev)方法处理事件。否则则交给子View处理，如果子View处理，则消费掉，如果子View不消费，返回false,交给更上级处理。

3.3 一些问题与注意事项

3.3.1 Activity、Window、View三者事件传递关系

  Activity-->Window-->View

3.3.2 OnTouchEvent(MotionEvent ev)与OnTouchListener(MotionEvent ev)的优先级

  如果设置了OnTouchListener(MotionEvent ev)并返回true，则不会调用OnTouchEvent(MotionEvent ev)，所以OnTouchListener(MotionEvent ev)优先级更高
