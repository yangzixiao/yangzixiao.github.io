Activity

1. 生命周期

  1. 在后台时被杀死重新打开时

    会重新创建activity，重走生命周期

  2. 按home键,再重新启动

    onPause - onStop
    onRestart - onStart - onResume

  3. 开启新的Activity

    该Activity onPause - 新Activity onCreate - onStart - onResume - 该Activity onStop

  4. 如果Activity配置configChanges orientation|screenSize
    activity屏幕旋转时则不会重建，而是会调用onConfigurationChanged方法
    不配置的话 系统会自动调用onSaveInstanceState保存当前Activity数据、状态，在onRestoreInstanceState方法中恢复数据

  5. onRestoreInstanceState调用时机
    onStart 与 onResume之间

2. 启动模式

  1. standard 标准模式
    即该种模式下的Activity每启动一次都会创建一个实例
  2. singleTop 栈顶复用模式
    该模式下，如果栈顶是该Activity，则会复用该Activity，并调用onNewIntent方法
    应用场景：解决重复点击造成的多次启动Activity问题
  3. singleTask 单任务模式
    该模式下，一个Task里只会存在相同的Actiity，如果重新启动该Activity，那么任务栈里该Activity上面的Activity都会出栈，如果在该Activity下指定新的android:taskAffinity属性，启动的时候回新建一个任务栈
  4. singleInstance
    该Activity单独占用一个任务栈

Activity、Window、View之间的关系

Activity相当于一个控制器，控制其生命周期和与处理事件，内部维护了了一个Window（PhoneWindow是其实现类）。
Window才是真正用来展示视图的，像一个装载器，通过WindowManager将DecorView加载其中，并交给RootView，进行视图绘制与交互
DecorView就是我们常说的根布局（其实是一个FrameLayout的子类），DecorView内部是一个LinearLayout，分为三个部分，上面是一个ViewStub，中间是一个Title，下面是一个Content（我们常说的setContentView就被加载到其中）

艺术探索笔记之View事件体系

发表于 2018-08-25

1. 基础知识

1.1 View的位置参数

1. top、left、right、bottom四个属性是相对于父控件
2. 3.0开始增加属性
    x ,y      view的左上角坐标
    translationX，translationY    view相对于左上角的偏移量

    与top、left、right、bottom关系如下
    x = left + translationX
    y = top + translationY

注意：view在平移过程中，top和left表示原始左上角的位置信息，其值不会变化，变化的是x,y，translationX,translationY这四个参数

1.2 MotionEvent 和 TouchSlop

1. MotionEvent

通过MotionEvent可以获取事件类型和事件坐标

事件类型:例如   ACTION_DOWN  ACTION_MOVE ACTION_UP

事件坐标:获取方式
getX(),getY() 获取到的坐标是相对于当前View的左上角坐标系
getRawX(),getRawY则是相对于屏幕左上角坐标系
