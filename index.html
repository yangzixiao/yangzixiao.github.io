App启动白屏问题
原因：
在Activity setContentView()之前，会先创建Window，这个时候没有就会使用默认的颜色，因此会导致出现白屏或黑屏

解决方案：给window设置背景图片或者透明颜色
<item name="android:windowBackground">@drawable/ic_splash</item>
or
<item name="android:windowBackground">@color/colorTransparent</item>

android笔记

发表于 2019-02-17

Activity

1. 生命周期

  1. 在后台时被杀死重新打开时

    会重新创建activity，重走生命周期

  2. 按home键,再重新启动

    onPause - onStop
    onRestart - onStart - onResume

  3. 开启新的Activity

    该Activity onPause - 新Activity onCreate - onStart - onResume - 该Activity onStop

  4. 如果Activity配置configChanges orientation|screenSize
    activity屏幕旋转时则不会重建，而是会调用onConfigurationChanged方法
    不配置的话 系统会自动调用onSaveInstanceState保存当前Activity数据、状态，在onRestoreInstanceState方法中恢复数据

  5. onRestoreInstanceState调用时机
    onStart 与 onResume之间

2. 启动模式

  1. standard 标准模式
    即该种模式下的Activity每启动一次都会创建一个实例
  2. singleTop 栈顶复用模式
    该模式下，如果栈顶是该Activity，则会复用该Activity，并调用onNewIntent方法
    应用场景：解决重复点击造成的多次启动Activity问题
  3. singleTask 单任务模式
    该模式下，一个Task里只会存在相同的Actiity，如果重新启动该Activity，那么任务栈里该Activity上面的Activity都会出栈，如果在该Activity下指定新的android:taskAffinity属性，启动的时候回新建一个任务栈
  4. singleInstance
    该Activity单独占用一个任务栈

Activity、Window、View之间的关系

Activity相当于一个控制器，控制其生命周期和与处理事件，内部维护了了一个Window（PhoneWindow是其实现类）。
Window才是真正用来展示视图的，像一个装载器，通过WindowManager将DecorView加载其中，并交给RootView，进行视图绘制与交互
DecorView就是我们常说的根布局（其实是一个FrameLayout的子类），DecorView内部是一个LinearLayout，分为三个部分，上面是一个ViewStub，中间是一个Title，下面是一个Content（我们常说的setContentView就被加载到其中）

内存泄漏

1. 概念：

生命周期长的对象持有生命周期短的对象导致内存无法回收

2. 常见内存泄漏问题

  1. 非静态内部类、匿名内部类

    原因：非静态内部类会引用外部类而且生命周期比外部类长
    解决方案：内部类静态化，静态内部类不持有外部类，生命周期与App一致

  2. 单例模式

    原因：单例的生命周期跟App相同
    解决方案：context最好用Application

  3. Handler

    原因：activity在关闭的时候可能会有延迟消息持有Activity
    解决方案：利用静态内部类和弱引用，再者就是在activity关闭的时候remove掉所有的消息
    注意：要做activity为空判断，否则可能会空指针

  4. list集合内存泄漏

    list持有对象未及时释放
    解决方案：在不用的地方list clear掉，置空

  5. 部分资源用完未关闭

    1. 广播未关闭
    2. 流未关闭
    3. webView
    4. 动画未关闭

3. 内存泄漏检测工具

  1. lint
  2. LeakCanary

Handler

1. 流程

创建Handler对象的时候，会初始化Looper对象，Looper对象里维护了一个MessageQueen。

Handler发送消息的时候，会把消息插入到Looper里维护的MessagerQueen里，在Handler发送消息的时候，会对Message对象做些赋值处理，例如Message的Target就是发送消息的Handler，再者就是携带一些数据之类的

而Looper loop()方法会不停的从MessageQueen里拿到Message,Message通过Message的target对象（也就是Handler），调用Handler的dispatchMessage()方法处理Message，dispatchMessage()方法在message的 callback(也就是Runnable,这其实是Handler发送消息的另种方式，Handler的post()方法)对象为空的时候，会调用Handler的handMessage()方法，也就是我们常用的处理Message信息的方法。

1. 基础知识

1.1 View的位置参数

1. top、left、right、bottom四个属性是相对于父控件
2. 3.0开始增加属性
    x ,y      view的左上角坐标
    translationX，translationY    view相对于左上角的偏移量

    与top、left、right、bottom关系如下
    x = left + translationX
    y = top + translationY

注意：view在平移过程中，top和left表示原始左上角的位置信息，其值不会变化，变化的是x,y，translationX,translationY这四个参数
